# RelationEcho

**情感陪伴 AI Agent | 记忆永不遗忘，关系自然生长**

一个能记住你、理解你、随相处而成长的 AI 陪伴伙伴。基于长期记忆、关系演化和人格一致性，提供真实的陪伴体验。

---

## 功能特性

- **🤖 核心聊天** — 双通道 Agent 循环（快速通道 + 完整通道）、行为化 Prompt、时间感知、多气泡分段发送
- **🧠 长期记忆系统** — LLM 驱动提取、两层标签体系（据实标签 + 记忆分类）、混合检索（向量 50% + 标签 30% + 权重 20%）、权重动态衰减与回升
- **👤 角色系统** — 50+ 字段三层人格结构（Part A/B/C）、LLM 文本创建、双默认角色（小雨/阿泽）、JSON 导入导出
- **📈 关系系统** — 0-4 级关系演化、行为指令影响 LLM 回复风格、初始等级推断、UI 徽章
- **💗 情绪系统** — 用户情绪分析 → 角色情绪共鸣（3 轮影响）
- **📋 用户画像** — 三层结构（与角色对称）、LLM 自动提取、可导出为角色
- **🔧 工具调用** — web_search / web_fetch（OpenAI function calling）、Tool Registry
- **🔗 记忆图谱** — d3-force 力导向图、SVG 渲染、按角色隔离
- **💬 多会话管理** — 微信式会话列表、记忆按角色隔离、摘要滚动队列（最多 10 条）

## 快速开始

```bash
# 克隆仓库
git clone https://github.com/xymumu-123/relation-echo.git
cd relation-echo

# 安装依赖
npm install

# 启动 Expo
npx expo start
```

> 首次启动会进入 Onboarding 引导页，生成角色后可开始对话。
> 需要先配置 API（支持 OpenAI 兼容 API）。

### API 配置

在「设置」页面配置：
- **LLM 模型**：任意的 OpenAI 兼容 API（如 GPT-4o、DeepSeek 等）
- **Embedding 模型**：可独立配置（留空则与聊天模型共用）
- 支持多配置管理、一键切换、连通性测试

## 架构概览

### 核心系统架构

```
用户输入
    │
    ▼
┌─────────────────────────────────────────────────┐
│              Agent Loop（agent-loop.ts）          │
│                                                   │
│  1. 危机检测 → 快速通道（fastPath）               │
│     - 情感关键词 → emotion 模式                   │
│     - 闲聊关键词 → chat 模式                      │
│     - 极端关键词 → 危机响应                       │
│                                                   │
│  2. 完整通道（fullPath）                          │
│     - 上下文组装（人格 + 记忆 + 关系 + 情绪）     │
│     - LLM 生成回复                                │
│     - 检查 tool_calls → 执行工具 → 再次 LLM       │
│     - reflect 阶段：记忆提取 / 摘要 / 画像更新    │
└─────────────────────────────────────────────────┘
    │
    ▼
  回复 + 记忆 + 摘要 → 存储
```

## 设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 记忆遗忘策略 | 永不遗忘，权重衰减 | 陪伴型 AI 不应遗忘旧事 |
| 标签体系 | 两层（据实 + 分类） | 兼顾"关于什么"和"什么性质"两个维度 |
| 混合检索 | 向量 50% + 标签 30% + 权重 20% | 不纯依赖向量相似度，引入语义分类和流行度 |
| 角色结构 | Part A/B/C 三层（50+ 字段） | 与用户画像对称，数据结构统一 |
| Prompt 风格 | 行为指令式（非声明式档案） | 减少 AI 味，LLM 更自然模仿人格 |
| 后台任务触发 | 数据库差值（非内存 modulo） | 重启后状态不丢失，更可靠 |
| 用户画像 | 全局共享（characterId = -1） | 所有角色对话贡献到同一画像 |
| 聊天 UI | Inverted FlatList | 标准聊天模式，无需滚动 hack |
